## Ubuntu 云服务器网卡配置丢失问题(cloud-init 覆盖)

**原因**:云平台预装的 cloud-init 服务在每次启动时会自动从云平台获取网络配置并覆盖手动修改,导致重启/重装后网卡配置丢失。

**解决思路**:让 cloud-init 不再管理网络配置。

---

### 1. 禁用 cloud-init 网络管理

```bash
sudo vim /etc/cloud/cloud.cfg.d/99-disable-network-config.cfg
```

写入:

```yaml
network: {config: disabled}
```

> 不确定能否编辑也可以用 `echo` 直接写入:
> 
> ```bash
> echo 'network: {config: disabled}' | sudo tee /etc/cloud/cloud.cfg.d/99-disable-network-config.cfg
> ```

---

### 2. 清理旧配置

删除 cloud-init 自动生成的 netplan 文件:

```bash
sudo rm /etc/netplan/50-cloud-init.yaml
```

---

### 3. 手动配置静态 IP

```bash
sudo vim /etc/netplan/01-netcfg.yaml
```

```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:               # 替换为你的网卡名称(ip a 查看)
      addresses:
        - 192.168.1.100/24
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        addresses:
          - 8.8.8.8
```

---

### 4. 应用并重启

```bash
sudo netplan apply
sudo reboot
```

---

### 💡 额外提醒:DNS 问题

如果能 ping 通 IP 但无法解析域名(如 `ping baidu.com` 失败),可能是 `systemd-resolved` 覆盖了 DNS:

- 优先方案:在上面的 netplan 文件里配置 `nameservers`(已包含)
- 或者直接禁用 `systemd-resolved` 服务

---

### 彻底卸载 cloud-init(可选)

如果完全不需要 cloud-init:

```bash
sudo apt purge cloud-init -y
sudo rm -rf /etc/cloud /var/lib/cloud
```

⚠️ **注意**:卸载后云平台将无法对实例做初始化操作(如密钥注入、自动扩容时的初始配置等)。对纯手动运维的场景影响不大,但如果用 AWS ParallelCluster 这类自动化编排平台(你之前在 `hermite-test`/`test0617` 上用过),节点初始化很依赖 cloud-init,**不建议卸载**,只做第 1 步禁用网络管理就够了。
