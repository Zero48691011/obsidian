# 私有云部署指南

## 概述

私有云（Private Cloud）是指在自有数据中心或机房中，用虚拟化/云管平台将物理服务器资源池化，对外提供类似公有云的 IaaS/PaaS 能力（自助申请 VM、对象存储、VPC 网络等）。

---

## 一、为什么需要私有云

| 场景 | 说明 |
|------|------|
| **数据合规** | 金融、医疗、政府等行业要求数据不出机房 |
| **成本控制** | 长期稳态负载下，自建比公有云便宜 30%~50% |
| **性能要求** | 裸金属性能、低延迟网络，公有云无法满足 |
| **混合云** | 核心业务跑私有云，弹性扩容跑公有云 |
| **技术自主** | 不被单一厂商锁定，掌握底层技术栈 |

## 二、主流私有云方案对比

### 方案总览

```
私有云方案
├── 开源全栈云平台
│   ├── OpenStack      ← 最成熟、生态最大、复杂
│   ├── CloudStack     ← Apache 项目、简单稳定
│   └── OpenNebula     ← 轻量、欧洲流行
│
├── 虚拟化管理平台
│   ├── VMware vSphere ← 企业标配、商业授权
│   ├── Proxmox VE     ← 开源、易上手、中小规模
│   └── oVirt (RHEV)   ← Red Hat 出品
│
├── 容器化/HCI 云平台
│   ├── Harvester      ← SUSE 出品、K8s + KubeVirt
│   └── Kubernetes + 插件 ← 自己组装
│
└── 超融合一体机
    ├── Nutanix        ← 商业超融合标杆
    └── 华为 FusionCube / 深信服 等
```

### 详细对比

| 方案 | 类型 | 规模 | 复杂度 | 优点 | 缺点 |
|------|------|:--:|:--:|------|------|
| **OpenStack** | 全栈云 | 大规模 | ⭐⭐⭐⭐⭐ | 功能最全、生态最大、社区活跃 | 部署运维复杂、组件多、升级痛苦 |
| **CloudStack** | 全栈云 | 中大规模 | ⭐⭐⭐ | 稳定、简单、Apache 基金会 | 生态较小、社区不如 OpenStack |
| **OpenNebula** | 全栈云 | 中小规模 | ⭐⭐⭐ | 轻量、单机可跑、VMware 集成好 | 国内用得少、社区小 |
| **VMware vSphere** | 虚拟化 | 各种规模 | ⭐⭐⭐ | 企业标准、稳定、生态完善 | 商业授权贵、被 Broadcom 收购后涨价 |
| **Proxmox VE** | 虚拟化 | 中小规模 | ⭐⭐ | 免费开源、Web UI 好用、支持 LXC+VM | 大规模集群能力弱、企业支持少 |
| **Harvester** | HCI/K8s | 中小规模 | ⭐⭐⭐ | 云原生、K8s 原生、SUSE 背书 | 较新（2021年）、生态还在成长 |
| **Kubernetes + 插件** | 容器云 | 各种规模 | ⭐⭐⭐⭐ | 灵活、云原生标准 | 需要自己组装、学习曲线陡 |

## 三、入门推荐：Proxmox VE

如果你只是想快速体验私有云，**Proxmox VE** 是最容易上手的选择。

### 3.1 是什么

Proxmox VE 是一个开源的虚拟化管理平台，基于 Debian，提供 Web 管理界面，支持：
- **KVM 虚拟机**（全虚拟化，可跑任何 OS）
- **LXC 容器**（轻量 Linux 容器）
- **Ceph 分布式存储**（内置集成）
- **集群管理**（多节点高可用、在线迁移）

### 3.2 硬件要求

| 组件 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 64 位 x86，支持 VT-x/AMD-V | 8 核以上 |
| 内存 | 2 GB | 32 GB+ |
| 系统盘 | 10 GB | 128 GB SSD |
| 数据盘 | — | 多块 SSD 做 Ceph/ZFS |
| 网卡 | 1 张 | 2 张（管理 + 业务分离） |

### 3.3 安装（超简单）

```bash
# 1. 下载 ISO
# https://www.proxmox.com/en/downloads

# 2. 制作 U 盘启动盘，像装 Linux 一样安装
# 安装过程中设置：
#   - 管理 IP（如 192.168.1.100）
#   - root 密码
#   - 磁盘分区（默认即可）

# 3. 装完后浏览器访问
https://192.168.1.100:8006
# 账号 root，密码为安装时设置的密码
```

### 3.4 创建第一台虚拟机

```
Web UI 操作流程：
上传 ISO 镜像 → 点击「创建虚拟机」→ 分配 CPU/内存/磁盘
→ 选择 ISO → 确认网络 → 启动 → 打开控制台安装系统
```

就这么简单，**不需要任何命令行操作**。

### 3.5 集群模式

Proxmox 支持多节点集群，实现高可用和在线迁移：

```bash
# 在节点1上创建集群
pvecm create my-cluster

# 在节点2上加入集群
pvecm add 192.168.1.100

# 查看集群状态
pvecm status
```

## 四、进阶方案：OpenStack

如果你需要**真正的公有云体验**（自助申请 VM、多租户、计量计费、VPC 网络），OpenStack 是事实标准。

### 4.1 OpenStack 核心组件

```
┌─────────────────────────────────────────────────────────┐
│                    OpenStack 架构                         │
│                                                          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │ Horizon  │  │  CLI/API │  │  SDK     │   ← 用户界面  │
│  │ (Web UI) │  │          │  │          │              │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘              │
│       └──────────────┼────────────┘                     │
│                      ▼                                   │
│  ┌─────────────────────────────────────┐                │
│  │           Keystone (认证)           │  ← 身份服务     │
│  └─────────────────────────────────────┘                │
│                      │                                   │
│  ┌───────┐ ┌───────┐ ┌───────┐ ┌──────────┐           │
│  │ Nova  │ │Neutron│ │Cinder │ │ Glance   │           │
│  │(计算) │ │(网络) │ │(块存储)│ │ (镜像)   │           │
│  └───────┘ └───────┘ └───────┘ └──────────┘           │
│  ┌───────┐ ┌───────┐ ┌───────┐                        │
│  │ Swift │ │ Heat  │ │Ceilometer│                      │
│  │(对象) │ │(编排) │ │ (计量)   │                      │
│  └───────┘ └───────┘ └───────┘                        │
└─────────────────────────────────────────────────────────┘
```

| 组件 | 作用 | 类比公有云 |
|------|------|-----------|
| **Keystone** | 认证授权、用户/项目/角色管理 | IAM |
| **Nova** | 虚拟机生命周期管理 | ECS / EC2 |
| **Neutron** | 虚拟网络、VPC、安全组、浮动 IP | VPC |
| **Cinder** | 块存储（云硬盘） | EBS / 云盘 |
| **Glance** | 虚拟机镜像管理 | 镜像服务 |
| **Swift** | 对象存储 | S3 / OSS |
| **Horizon** | Web 管理界面 | 控制台 |
| **Heat** | 基础设施即代码（模板编排） | ROS / CloudFormation |

### 4.2 部署方式

| 方式 | 适用场景 | 难度 |
|------|----------|:--:|
| **DevStack** | 学习/开发，单机 All-in-One | ⭐ |
| **Kolla-Ansible** | 生产环境，容器化部署 | ⭐⭐⭐ |
| **OpenStack-Ansible** | 大规模生产 | ⭐⭐⭐⭐ |
| **TripleO** | Red Hat 官方方案 | ⭐⭐⭐⭐ |

### 4.3 DevStack 快速体验（学习用）

```bash
# 准备一台 Ubuntu 22.04 机器（至少 8GB 内存、4 核）

# 创建 stack 用户
sudo useradd -s /bin/bash -d /opt/stack -m stack
echo "stack ALL=(ALL) NOPASSWD: ALL" | sudo tee /etc/sudoers.d/stack
sudo su - stack

# 克隆 DevStack
git clone https://opendev.org/openstack/devstack
cd devstack

# 创建最小化配置
cat > local.conf << 'EOF'
[[local|localrc]]
ADMIN_PASSWORD=secret
DATABASE_PASSWORD=$ADMIN_PASSWORD
RABBIT_PASSWORD=$ADMIN_PASSWORD
SERVICE_PASSWORD=$ADMIN_PASSWORD
HOST_IP=192.168.1.100
EOF

# 开始安装（约 30~60 分钟）
./stack.sh

# 安装完成后访问
# http://192.168.1.100/dashboard
# 账号 admin，密码 secret
```

### 4.4 Kolla-Ansible 生产部署（概要）

```bash
# 1. 准备多台服务器（控制节点 + 计算节点 + 存储节点）
# 2. 安装 Docker + Ansible
# 3. 配置 multinode
cat > /etc/kolla/globals.yml << 'EOF'
kolla_base_distro: "ubuntu"
kolla_install_type: "source"
network_interface: "eth0"
neutron_external_interface: "eth1"
kolla_internal_vip_address: "192.168.1.250"
enable_cinder: "yes"
enable_swift: "no"
EOF

# 4. 生成密码
kolla-genpwd

# 5. 部署
kolla-ansible -i /etc/kolla/multinode deploy

# 6. 初始化
kolla-ansible post-deploy
```

## 五、容器化方案：Harvester + Kubernetes

如果你想用**云原生**方式做私有云，Harvester 是当前最值得关注的方案。

### 5.1 Harvester 架构

```
Harvester 节点
├── K3s（轻量 Kubernetes）
├── KubeVirt（在 K8s 中管理 VM）
├── Longhorn（分布式块存储）
└── Multus（多网络支持）
```

把 VM 当成 K8s 的 Pod 来管理，用 `kubectl` 就能创建虚拟机。

### 5.2 安装

```bash
# 下载 ISO
# https://github.com/harvester/harvester/releases

# 像装 Linux 一样安装，选 "Create Harvester Cluster"
# 安装完成后，浏览器访问管理 IP，上传 license 即可使用
```

### 5.3 创建 VM

Harvester 提供 Web UI，也可以写 YAML 声明式创建：

```yaml
apiVersion: kubevirt.io/v1
kind: VirtualMachine
metadata:
  name: my-vm
spec:
  running: true
  template:
    spec:
      domain:
        cpu:
          cores: 2
        devices:
          disks:
          - disk:
              bus: virtio
            name: rootdisk
        memory:
          guest: "4Gi"
      volumes:
      - name: rootdisk
        persistentVolumeClaim:
          claimName: my-vm-disk
```

## 七、常见问题

**Q: 私有云和虚拟化有什么区别？**
> 虚拟化（如 Proxmox、VMware）只是把物理机分成多个虚拟机。私有云在虚拟化之上增加了**自助服务、多租户、计量计费、编排、API 驱动**等能力。简单说：私有云 = 虚拟化 + 云管平台。

**Q: 最小私有云要几台机器？**
> 学习/测试：**1 台**（Proxmox 或 DevStack）。生产环境：至少 **3 台**（控制节点）+ N 台计算节点。

**Q: 运维私有云需要多少人？**
> 小型（< 20 台）：1~2 人兼职。中型（20~100 台）：2~3 人专职。大型（100+ 台）：5+ 人团队。

**Q: 私有云和公有云能打通吗？**
> 可以。通过专线/VPN 打通网络，Kubernetes 联邦、统一监控等。这叫**混合云（Hybrid Cloud）**。

**Q: 用 K8s 部署应用算私有云吗？**
> 不算。K8s 是容器编排平台，不是 IaaS 层。但如果加上 KubeVirt（管理 VM）、MetalLB（LB）、OpenEBS/Longhorn（存储）、Istio（网络），就接近私有云的能力了。Harvester 就是这种做法。
