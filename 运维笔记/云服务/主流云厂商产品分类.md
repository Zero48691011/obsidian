# 主流云厂商产品分类对比

## 概述

本文档梳理四大主流云厂商（AWS、Azure、Google Cloud、阿里云）的核心产品线，按功能类别横向对比，帮助快速建立云服务全景认知。

---

## 一、厂商概览

| 厂商 | 全球份额 | 国内份额 | 特点 |
|------|:------:|:------:|------|
| **AWS** | ~32% | — | 产品最全、生态最成熟、文档最丰富 |
| **Azure** | ~23% | — | 企业 Microsoft 生态集成、混合云强 |
| **Google Cloud** | ~11% | — | 数据/AI/容器技术领先（K8s 发明者） |
| **阿里云** | ~5% | ~35% | 国内第一、双11 验证、中文生态完善 |

> 国内还有华为云、腾讯云、百度云等，本文以阿里云代表国内阵营。

## 三、产品命名规律

各厂商的命名风格有迹可循，有助于快速猜出陌生产品是干什么的：

| 厂商 | 命名风格 | 示例 |
|------|----------|------|
| **AWS** | 直白英文缩写 | EC2 = Elastic Compute Cloud, S3 = Simple Storage Service |
| **Azure** | 功能描述型 | Virtual Machines, Blob Storage, Functions |
| **Google Cloud** | 功能描述 + 统一前缀 | Cloud Storage, Cloud SQL, Cloud Run |
| **阿里云** | 缩写 + 中文 | ECS = 云服务器, OSS = 对象存储, ACK = 容器服务 |

## 四、选型建议

### 场景导向

| 场景 | 推荐 | 理由 |
|------|------|------|
| **出海/全球化** | AWS | 全球 Region 最多、生态最成熟 |
| **微软技术栈（.NET/SQL Server）** | Azure | 原生集成、混合云授权优势 |
| **数据/AI 密集型** | Google Cloud | BigQuery + Vertex AI 数据链路最短 |
| **国内业务** | 阿里云 | 国内基础设施最完善、中文文档齐全 |
| **容器/K8s 优先** | Google Cloud | K8s 发明者，GKE 体验最好 |
| **成本敏感** | 阿里云 / AWS Spot | 阿里云有竞价实例，AWS Spot 折扣大 |

### 多厂商策略

大中型企业通常采用 **多云** 或 **混合云** 策略：
- 核心业务跑一家主力云，灾备或特定场景用第二家
- 避免厂商锁定，用 Terraform / Pulumi 做多云 IaC
- 跨云网络互联用专线或 SD-WAN
