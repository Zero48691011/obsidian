K8s 集群 + 中间件 架构拓扑

3 台物理机 · 2 个 K8s 工作节点 · 1 个中间件节点

K8s 工作节点 (×2)

  • • 运行 8 个微服务 Pod（每节点 4 个）
  • • kubelet + kube-proxy + containerd
  • • Calico CNI 组网，Pod CIDR 互通
  • • CoreDNS 集群内 DNS 解析
  • • Deployment 管理副本，滚动更新

中间件节点

  • • MySQL — 关系型存储，主从读写分离
  • • Redis — 分布式缓存 / 会话 / 锁
  • • RabbitMQ — 异步消息解耦
  • • MongoDB — 文档存储 / 日志归档
  • • Nacos — 服务注册发现 & 配置管理

流量路径

  • • 客户端 → Ingress (Nginx) :80/:443
  • • Ingress → K8s Service → Pod
  • • Pod → Nacos (服务发现)
  • • Pod → MySQL / Redis / RMQ / Mongo
  • • 中间件通过内网 IP 直连，不走 Ingress
